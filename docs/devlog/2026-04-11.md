# Дневник разработки — go-kpi-tenders

## 2026-04-11 — День 1: Scaffolding и рефакторинг

### Что сделано

**1. Базовый scaffolding (8 шагов)**
- `.env.example` — секции App, DB, Redis, S3 с дефолтами под локальный docker-compose
- `internal/config/config.go` — парсинг через `cleanenv`, `.env` читается опционально, приоритет у ENV
- `pkg/logging/logging.go` — `slog`: TextHandler для local, JSONHandler для prod
- `docker-compose.yml` — PostgreSQL (`pgvector/pgvector:pg16`), Redis 7, MinIO
- SQL миграция `000001_init_schema` — 5 таблиц: organizations, users, projects, documents, document_tasks + расширения uuid-ossp и pgvector
- `sqlc.yaml` — pgx/v5, overrides для uuid→google/uuid, timestamptz→time.Time, vector→pgvector.Vector
- `sql/query/document.sql` — CRUD для documents и document_tasks
- Health check handler + 3 теста (httptest + testify)
- `cmd/api/main.go` — config → logger → pgxpool → server → graceful shutdown
- `Makefile` — up, down, createdb, dropdb, migrate_up, migrate_down, sqlc (Docker), test, run

**2. Рефакторинг: паттерн Server Struct**
- Создан `internal/server/server.go` — структура `Server` с config, logger, repo, db, router, services
- `NewServer()` настраивает Gin, CORS (разные правила для local/prod), регистрирует маршруты
- Группы маршрутов:
  - `GET /health` — liveness probe
  - `/internal/worker/*` — service-to-service (ServiceBearerAuth middleware)
  - `/api/v1/health` — публичный
  - `/api/v1/documents/*`, `/api/v1/tasks/*` — защищённые (AuthMiddleware)
- Middleware-заглушки: AuthMiddleware, ServiceBearerAuth
- Handler stubs для documents и tasks
- `internal/service/document.go` — DocumentService → repository.Queries
- `cmd/api/main.go` стал тонким: только инициализация и graceful shutdown

**3. Реструктуризация на каноничный Go layout**
- `cmd/internal/` → `internal/` (корень)
- `cmd/pkg/` → `pkg/` (корень)
- `cmd/main/app.go` → `cmd/api/main.go`

**4. Настройка pgvector и sqlc**
- Docker-образ заменён на `pgvector/pgvector:pg16`
- sqlc переведён на Docker-запуск (`sqlc/sqlc:1.30.0`)
- Overrides: uuid→google/uuid, timestamptz→time.Time, vector→pgvector.Vector
- `emit_empty_slices: true` — LIST запросы возвращают `[]` вместо `null` в JSON

### Стек

### Решения и обоснования
- **google/uuid вместо pgtype.UUID** — чистый JSON маршалинг (`"uuid-string"` вместо `{"Bytes":...,"Valid":true}`), сравнение через `==`, стандарт Go-экосистемы
- **time.Time вместо pgtype.Timestamptz** — нативный Go-тип, удобнее в бизнес-логике
- **sqlc через Docker** — консистентность версий у всех разработчиков и на CI
- **emit_empty_slices** — фронтенд получает `[]` вместо `null`
