### Что сделано

**1. Новый пакет `internal/store` — Store-интерфейс**

Выделен пакет `internal/store` с интерфейсом `Store` и реализацией `SQLStore`:

- `Store` расширяет `repository.Querier`, добавляя метод `ExecTx(ctx, fn func(Querier) error) error`
- `SQLStore` — production-реализация поверх `*pgxpool.Pool`: открывает pgx-транзакцию, передаёт `*repository.Queries` обёрнутый в `WithTx`, при панике откатывает
- `store.New(pool)` — конструктор `SQLStore`; сценарии без БД обрабатываются на уровне `server.NewServer`, который создаёт store только при наличии пула

Создан `internal/store/mock/mock_store.go` — testify-мок для `Store`:
- `MockStore.ExecTx` вызывает переданный `fn(m)` напрямую, если на метод не настроена ошибка
- Совместим с testify `mock.On` / `mock.AssertExpectations`

### Принятые решения

- **`bcrypt` до транзакции** — стандартная практика: держать транзакцию максимально короткой; хеширование паролей (100–300 мс) не должно конкурировать за соединение из пула
- **`Store` расширяет `Querier`** — сервисы без транзакций (Auth, Document) принимают `Querier`; сервисы с транзакциями (Organization) принимают `Store` — минимальная поверхность зависимости
- **`MockStore.ExecTx` вызывает `fn(m)`** — мок транзакции должен проверять поведение внутри колбека, а не только факт вызова `ExecTx`
- **`pool == nil → store == nil`** — позволяет создавать `Server` в тестах без настоящей БД

**🔴 Python-worker endpoint отсутствует**
В `server.go` есть `// TODO: register Python worker endpoints`. Группа `/internal/worker` содержит только `/ping`.
Требуется: `PATCH /internal/worker/tasks/{id}/status` — обновление статуса задачи воркером.

**🟡 Незафиксированная размерность вектора**
`catalog_positions.embedding` объявлен как `vector` без размерности в `000002_catalog_positions.up.sql`.
Когда будет выбрана модель эмбеддингов — потребуется миграция `ALTER COLUMN embedding TYPE vector(N)`.
Кандидаты: `vector(1536)` (OpenAI text-embedding-3-small/large), `vector(1024)` (Cohere).

- `OrganizationService.Update()` и `.Delete()` не покрыты
