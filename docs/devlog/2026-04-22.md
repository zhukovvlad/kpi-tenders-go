# Дневник разработки — go-kpi-tenders

## 2026-04-22 — День 6: GET /api/v1/auth/me, изоляция профиля по тенанту

### Что сделано

**1. SQL-запрос `GetUserByIDAndOrg`**

Добавлен новый запрос в `sql/query/user.sql`:

```sql
-- name: GetUserByIDAndOrg :one
SELECT id, organization_id, email, full_name, role, is_active, created_at, updated_at
FROM users
WHERE id = $1 AND organization_id = $2;
```

Запрос намеренно исключает `password_hash` — возвращаемые данные полностью безопасны для отправки клиенту. Условие `AND organization_id = $2` гарантирует tenant-изоляцию: пользователь не может получить профиль из чужой организации.

После добавления запроса выполнены `make sqlc` и `make mock` — сгенерированы `GetUserByIDAndOrgRow`, `GetUserByIDAndOrgParams` и обновлён `MockQuerier`.

**2. Метод `UserService.GetProfile`**

Добавлен в `internal/service/service_user.go`:

- Принимает `userID` и `orgID` (оба `uuid.UUID`)
- `pgx.ErrNoRows` → `errs.CodeNotFound` с сообщением `"user not found"`
- `is_active == false` → `errs.CodeUnauthorized` с нейтральным сообщением `"account is unavailable"` (не раскрывает причину деактивации)
- Любая другая ошибка БД → `errs.CodeInternalError`

**3. Хендлер `Server.GetMe`**

Добавлен в `internal/server/handler_user.go`:

- Извлекает `userID` и `orgID` через `s.userIDFromContext` / `s.orgIDFromContext`
- Вызывает `s.userService.GetProfile`
- Возвращает JSON без `password_hash` и без служебных полей (`created_at`, `updated_at`):

```json
{
  "id": "uuid",
  "organization_id": "uuid",
  "email": "user@example.com",
  "full_name": "Иван Иванов",
  "role": "admin",
  "is_active": true
}
```

**4. Роутинг**

В `internal/server/server.go` в группу `auth` добавлен маршрут:

```go
auth.GET("/me", s.AuthMiddleware(), s.GetMe)
```

`AuthMiddleware()` применён per-route, а не на всю группу — остальные маршруты (`/register`, `/login`, `/refresh`, `/logout`) остаются публичными.

### Принятые решения

- **`organization_id` в WHERE** — запрос профиля обязан содержать тенант-условие, иначе любой авторизованный пользователь мог бы получить профиль пользователя из другой организации по `id` (IDOR, OWASP A01)
- **Нейтральное сообщение при деактивации** — `"account is unavailable"` вместо `"account is deactivated"` не даёт атакующему понять, что аккаунт существует, но заблокирован
- **Per-route middleware** — добавление `s.AuthMiddleware()` только на `/me` вместо перевода всей группы `/auth` в protected-зону сохраняет публичность публичных маршрутов без создания второй группы
- **`newServerWithMockUserService`** — хелпер подменяет только `userService`, оставляя живым `authService` с реальными JWT-секретами; тест генерирует настоящий токен и проверяет полный путь запроса
