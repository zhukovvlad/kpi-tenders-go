Новые таблицы для хранения ключей экстракции и извлечённых значений:

```sql
CREATE TABLE extraction_keys (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    organization_id UUID NOT NULL REFERENCES organizations(id) ON DELETE CASCADE,
    key_name        TEXT NOT NULL,
    source_query    TEXT NOT NULL,
    data_type       TEXT NOT NULL DEFAULT 'text',
    created_at      TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT now(),
    UNIQUE (organization_id, key_name)

CREATE TABLE extracted_data (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    organization_id UUID NOT NULL REFERENCES organizations(id) ON DELETE CASCADE,
    document_id     UUID NOT NULL REFERENCES documents(id) ON DELETE CASCADE,
    key_id          UUID NOT NULL REFERENCES extraction_keys(id) ON DELETE CASCADE,
    extracted_value TEXT,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT now(),
    UNIQUE (document_id, key_id)
);
```

Также добавлена колонка `md_document_id UUID REFERENCES documents(id)` в `document_tasks` — ссылка на MD-артефакт, по которому строится экстракция.

### `internal/service/service_extraction.go`

```go
type ExtractionService struct { q repository.Querier; pythonClient workerPythonClient; log *slog.Logger }

func (s *ExtractionService) Initiate(ctx, docID, orgID uuid.UUID, questions []string) (DocumentTask, error)
```

**Логика `Initiate`:**

1. Валидация: `len(questions) == 0` → `validation_failed`
2. `GetDocument(docID, orgID)` — проверка владельца
3. `ListExtractionKeysByOrg(orgID)` — получить существующие ключи организации
4. `CreateDocumentTaskInternal(module="resolve_keys", inputStoragePath=doc.StoragePath)` — создать задачу; при `pgx.ErrNoRows` → `conflict`
5. `pythonClient.Process(...)` с kwargs `raw_questions` + `existing_keys` (best-effort, ошибки логируются)

### `internal/service/service_worker.go` — новые методы

#### `triggerExtract(ctx, resolveTask)`

Вызывается из `HandleStatusUpdate` при `module=resolve_keys, status=completed`:

1. Парсит `resolveKeysPayload` из `result_payload`
2. `GetDocumentByID(docID)` — исходный документ
3. `UpsertExtractionKey` для каждого нового ключа из payload (best-effort per-key)
4. `GetDocumentByID(mdDocID)` — MD-артефакт из payload
5. `CreateDocumentTaskInternal(module="extract", inputStoragePath=mdDoc.StoragePath)`
6. `Process(...)` с kwargs `extraction_schema` + `md_document_id`

#### `handleExtractCompleted(ctx, task)`

Вызывается при `module=extract, status=completed`:

1. Парсит `map[string]string` из `result_payload`
2. `GetDocumentByID` — для получения `OrganizationID`
3. `GetExtractionKeysByNames(keyNames, orgID)` — получить ключи по именам
4. `UpsertExtractedDatum` для каждой пары ключ/значение (log-and-continue)

#### Новые типы payload

```go
type resolveKeysPayload struct {
    NewKeys []resolveKeysNewKey    `json:"new_keys"`
    Schema  []extractionSchemaEntry `json:"extraction_schema"`
    MDDocID string                  `json:"md_document_id"`
}
```

## Текущее состояние API

```text
POST /api/v1/documents/:id/extract  — запуск семантической экстракции
```

Полная цепочка: `resolve_keys` (определяет/пополняет ключи) → `extract` (заполняет `extracted_data`) → callback `PATCH /internal/worker/tasks/:id/status`.
